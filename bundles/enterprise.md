## What each skill does

- **brand-guidelines** — Applies your brand colours, fonts, and tone to everything produced. Edit `enterprise/brand-guidelines.md` with your actual brand details before using.
- **internal-comms** — Writes any internal business communication using the right template.
- **verify-before-versioning** — Before creating any new skill file, or a new version of an existing skill/bundle, always do a fresh, live search of the actual Drive folder right now. Always-on.
- **trust-the-live-signal** — When a stored value or documentation disagrees with a live, directly-checkable signal, trust the live signal. Always-on. Useful here whenever quoting a business figure — prefer the current live number over a remembered one when both are available.
- **no-assumed-memory** — Never assume a fresh session shares memory with a different conversation. Always-on whenever work crosses a session boundary.
- **propagate-the-lesson** — UPDATED to v1.1. Whenever a genuinely general-purpose lesson or fix is identified, check every bundle for the same applicability. Now also explicitly checks whether a new skill needs registering in comprehensive-audit's own lists, not just added to a bundle. Always-on.
- **close-known-gaps** — Once a real, known issue has been surfaced, the default is to fix it now rather than narrowing scope. Always-on.
- **board-of-directors** — Convenes the user's personal Board of Directors. Say "convene my board" at any point.

## How to trigger each skill

- "Apply our brand to this" / "Make this on-brand" → brand-guidelines
- "Write a status update" / "Draft an email to leadership" → internal-comms
- "Convene my board" → board-of-directors
- verify-before-versioning, trust-the-live-signal, no-assumed-memory, propagate-the-lesson, and close-known-gaps trigger automatically.
