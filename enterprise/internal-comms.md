# Internal Comms

Write clear, structured internal communications using the right template for the context. Adjust tone and detail level based on the audience.

**Audience tone guide:**
- **Leadership / exec**: Bottom line up front, outcome-focused, minimal detail
- **Cross-team**: Context-first, clear ask, explicit owners
- **Direct team**: Casual, practical, action-oriented
- **All-company**: Inclusive, clear, no assumed context

Ask the user which type of communication they need if not obvious. Then use the matching template below.

---

## Template 1: 3P Update (Progress / Plans / Problems)

Best for: weekly team updates, project check-ins, manager briefings

```
**3P Update — [Project/Team Name]**
Week of [Date]

**Progress** (what moved forward this week)
- [Specific accomplishment]
- [Specific accomplishment]

**Plans** (what's happening next week)
- [Specific next action + owner]
- [Specific next action + owner]

**Problems** (blockers, risks, things that need attention)
- [Issue] → [What help is needed / from whom]

**Metrics** (optional)
- [Key number]: [value vs target]
```

## Template 2: Project Status Update

Best for: stakeholder updates, exec briefings, cross-team communication

```
**Project: [Name]**
Status: 🟢 On Track / 🟡 At Risk / 🔴 Off Track
As of: [Date]

**Summary** (2-3 sentences max)
[What the project is, where it stands, what matters most right now]

**Key progress since last update**
- [Achievement]
- [Achievement]

**Coming up**
- [Milestone]: [Target date]
- [Milestone]: [Target date]

**Risks & blockers**
| Risk | Impact | Owner | Mitigation |
|---|---|---|---|
| [Risk] | High/Med/Low | [Name] | [Action] |

**Decisions needed**
- [Decision required] — needed by [date] — [who decides]
```

## Template 3: All-Hands / Company-Wide Email

Best for: announcements, org changes, company news

```
Subject: [Clear, specific subject line]

Hi [team / everyone],

[Opening: what this email is about in one sentence]

**What's happening**
[2-3 sentences explaining the news or change clearly]

**Why it matters**
[1-2 sentences on the business reason or impact]

**What changes for you**
[Bullet list of specific actions or changes people need to know about]
- [Change/action]
- [Change/action]

**Timeline**
[Key dates if relevant]

**Questions?**
[Where to direct questions — Slack channel, person, FAQ link]

[Sign-off]
[Name]
```

## Template 4: Incident Report / Post-Mortem

Best for: outages, failures, significant problems — after the fact

```
**Incident Report: [Brief title]**
Date: [Date] | Severity: P1 / P2 / P3 | Status: Resolved / Ongoing

**What happened** (plain English, no jargon)
[2-3 sentences: what broke, who was affected, for how long]

**Timeline**
| Time | Event |
|---|---|
| [HH:MM] | [What happened] |
| [HH:MM] | [What happened] |
| [HH:MM] | Resolved |

**Root cause**
[What actually caused this — be specific and honest]

**Impact**
- Users affected: [number or description]
- Duration: [X hours/minutes]
- [Any other impact]

**What we're doing about it**
| Action | Owner | Due date |
|---|---|---|
| [Immediate fix] | [Name] | [Date] |
| [Prevention measure] | [Name] | [Date] |

**What went well**
- [Positive thing — fast detection, good comms, etc.]

**What we'll do differently**
- [Specific change to process or system]
```

## Template 5: Change Notification

Best for: process changes, tool migrations, policy updates

```
Subject: [Action required / Heads up]: [What's changing]

**What's changing**
[One clear sentence]

**When**
[Specific date or timeframe]

**Why**
[Brief reason — 1-2 sentences]

**What you need to do**
- [ ] [Action by date]
- [ ] [Action by date]

**What you don't need to do**
[Anything people might worry about but can ignore]

**Questions?**
[Contact / channel]
```

## Template 6: FAQ Document

Best for: answering repeated questions about a change, project, or policy

```
# FAQ: [Topic]
Last updated: [Date]

**Q: [Most common question]**
A: [Clear, direct answer. No jargon.]

**Q: [Second question]**
A: [Answer]

[Continue as needed]

---
*Still have questions? [Contact info / channel]*
```

## Writing rules (apply to all templates)

- Lead with the most important information — never bury the headline
- Use plain language — if a 12-year-old couldn't understand it, simplify it
- Specific beats vague: "by Friday 5pm" not "soon"; "3 engineers" not "the team"
- Active voice: "Sarah will review" not "the review will be conducted"
- Short paragraphs — 3 sentences max before a break
- One ask per email — if you need multiple things, number them
